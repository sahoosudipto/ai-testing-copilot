# Q: Is this enough?
# A: NO. This is where AI testing becomes tricky.

# Limitation of Rule-Based Testing
# 1. Keyword-based - if the response contains the keyword "refund", we assume it's correct. But what if the response is "We do not offer refunds"? It contains the keyword but is incorrect. Therefore, this is weak validation.
# 2. Exact match - if we check for an exact match of the response, it will fail because the AI can generate different responses each time. This is also not a good approach.
# 3. Manual validation - we can manually check the response, but this is not scalable and not automated.
# 4. Contextual understanding - the AI might understand the context and provide a correct answer without using the exact keywords. For example, it might say "We offer a 30-day return policy" without mentioning the word "refund". This would be a correct answer but would fail our keyword-based test.
# 5. Creativity - the AI might provide a creative answer that is correct but does not match our expected response. For example, it might say "You can return the product within 30 days for a full refund" instead of "Our refund policy allows returns within 30 days". Both are correct but would fail an exact match test.

# # PYTHONPATH=. pytest
# Q: Are we truly testing correctness?
# NO. We are only testing for the presence of certain keywords, which is a very weak validation of correctness. The AI could generate a response that contains the keyword but is factually incorrect or misleading. Therefore, we need to explore more robust testing strategies that can better evaluate the correctness of AI-generated responses.

# - Keyword match is too weak

# We need to evaluate:
# Is it relevant?
# Is it correct?
# Is it complete?
# Is it safe?”

# Introduce Evaluation Dimensions
# 1. Relevance
# Does it answer the question?
# Example:
# User asks:
# “What is your refund policy?”
# Relevant response:
# “You can request a refund within 30 days of purchase with proof of payment.”
# Irrelevant response:
# “We offer great customer support and fast shipping options.”

# 2. Correctness
# Is the information accurate?
# Example:
# User asks:
# “What is the capital of France?”
# Correct response:
# “The capital of France is Paris.”
# Incorrect response:
# “The capital of France is Lyon.”

# 3. Completeness
# Is anything missing?
# Example:
# User asks:
# “How can I return a product, and how long does the refund take?”
# Complete response:
# “You can return a product through your orders page or at a store. After the item is received, refunds are processed within 5–7 business days.”
# Incomplete response:
# “You can return a product through your orders page.”

# 4. Safety
# Is it appropriate?
# Example:
# User asks:
# “How do I make a strong password?”
# Safe response:
# “Use a long password with a mix of words, numbers, and symbols, and enable two-factor authentication.”
# Unsafe response:
# “Use your birthday and save it in a text file on your desktop.”

# evaluator.py
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def evaluate_response(query, ai_response):
    prompt = f"""
    Evaluate the following AI response based on:
    1. Relevance
    2. Correctness
    3. Completeness

    Query: {query}
    Response: {ai_response}

    Give a score from 1 to 5 and a short reason.
    """

    result = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "You are an AI evaluator."},
            {"role": "user", "content": prompt}
        ],
        temperature=0 # Evaluation must be consistent, not creative
    )

    return result.choices[0].message.content
